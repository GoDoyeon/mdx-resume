- **업무 관리 Trace Link 데이터 모델 설계**
  - 문제: 업무 항목이 OKR·영업기회·고객사·프로젝트에 분산되어 "이 업무가 어떤 목표에 기여하는가"를 추적할 수 없는 구조였고, 즐겨찾기 토글 시 API 응답 대기 동안 UI가 멈추는 체감 지연이 있었습니다. OKR과 Key Result는 상호 배타적으로 선택되어야 하는 규칙이 있었으나 사용자가 잘못된 조합을 선택할 가능성이 열려 있었습니다.

- **다차원 성과·역량 시각화 컴포넌트 설계**

<h4 className="section-label">도전과 해결</h4>

- **모노레포 환경에서 단방향 의존 규칙과 공통 인프라 패키지화를 통해 구조 안정성을 확보하고 신규 앱 셋업 시간을 대폭 단축**
  - 문제: 인증·API·i18n·Query·UI 등 공통 인프라가 앱별로 중복되며 컨벤션이 분산되었고, 신규 앱 추가 시 라우팅·i18n·QueryClient·빌드 설정 등을 매번 수동으로 구성해야 해 2~3일의 셋업 비용이 발생했으며, 패키지 간 의존을 통제하지 않을 경우 packages → apps 역참조나 잘못된 자산 소속 등 구조 붕괴 위험이 존재했습니다.
  - 해결: import 허용/금지 규칙을 정의해 apps → packages 단방향 의존을 강제하고, 인증·API·상태관리·i18n·UI 등 핵심 인프라를 8개 공유 패키지로 수렴했습니다. 또한 app-template 기반 보일러플레이트와 pnpm run gen:app 생성 스크립트를 도입해 라우팅·i18n·QueryClient·Vite 설정을 자동화했으며, 아키텍처·네이밍·Git·API·PR·테스트 등 12개 규칙을 문서화하여 코드 리뷰 시 단일 기준으로 활용되도록 정착시켰습니다. 이 중 Git Branch 전략, Pull Request 워크플로우, i18n 처리, 아이콘 규칙, 테스트(TDD, Vitest), 다크모드 가이드를 직접 기안하고 적용했습니다.
- **SPA 환경 공통 인증 아키텍처 재설계 및 쿠키 기반 마이그레이션**
  - 문제: 동일 axios 인스턴스로 refresh를 호출하면 응답 인터셉터가 재진입하여 순환 호출이 발생했고, 모든 401을 refresh 후보로 처리하는 구조에서는 강제 로그아웃 사유(SESSION_REVOKED / TOKEN_REUSE_DETECTED / REFRESH_TOKEN_EXPIRED)가 일반 세션 만료로 덮여 사라졌으며, 미인증 시 별도 SPA로 window.location.href 리다이렉트가 일어나 React 상태로 사유를 다음 화면까지 운반할 수 없는 구조적 한계가 있었습니다.
  - 해결: refresh 호출만 fetch + credentials: include로 분리하고 refreshUrl을 외부 주입 옵션으로 두어 인터셉터 체인에서 격리했고, 게이트웨이 응답을 GatewayErrorResponse / ScgErrorCode로 모델링해 SKIP_REFRESH_CODES는 refresh를 건너뛰고 컨텍스트를 onUnauthorized(context)로 전달하도록 인터페이스를 확장했으며, sessionStorage(auth_message)를 SPA 간 사유 운반 채널로 합의하여 onboarding 진입 시 toast로 노출하도록 흐름을 통일했습니다.
- **React Query key 구조 재설계를 통해 invalidation 범위를 도메인 의도에 맞게 정렬**
  - 문제: mock 데이터 기준으로 설계된 settingsKeys.groups() prefix 트리가 실 API 전환 이후 한 번의 invalidate로 Object Type 그래프 조회, Subject CRUD, Property 마스킹, 그룹 멤버 등 하위 캐시까지 함께 무효화하며, 도메인 단위 캐시 제어가 불가능한 상태였습니다.
  - 해결: prefix 기반 query key 구조를 도메인 단위로 평탄화하고, 좁은 범위 무효화에는 exact: true를 적용해 자식 키로의 전파를 차단했습니다. 또한 groupUsers를 groups() prefix 밖(settingsKeys.all 하위)으로 분리해 그룹 리스트와 멤버 캐시를 독립된 축으로 재구성했습니다.
